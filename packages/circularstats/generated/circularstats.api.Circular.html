circularstats.api.Circular¶

class circularstats.api.Circular(low=-3.1415926535897931, upp=3.1415926535897931, units='radians', modulo=True, rad_low=-3.1415926535897931, rad_upp=3.1415926535897931, style='angle')¶

Class to keep track of circular data

mainly transformation to radians

Parameters :

low, upp : float

bounds of interval, used for modulo and integration. defines also the units of the data

units : {radians or any other string}

If radians, then the data is already assumed to be in radians. TODO: looks either redundant or not used enough, see Notes

modulo : bool

If True (default), then conversion of units includes taking modulo, it maps data into the defined interval

rad_low, rad_upp : float

Interval used for radians, common are either (-pi, pi) or (0, 2*pi)

style : string or tuple

style options for polar plots. If it is a tuple of two floats, then those are interpreted as offset and direction. Direction is -1 for clockwise and 1 for counter-clockwise. Strings can be used for common styles, ‘angle’ = (0, 1), ‘clock’ = (pi/2, -1), ‘geographic’=(pi/2, 1)

Notes

radians to radians conversion is used to shift the origin for modulo and integration, e.g. cdf of a circular distribution. This is not fully implemented yet.

The style option, offset and orientation, are used only for plotting, they are not used to define the origin and orientation of the transformations. The orientation is always assumed to be counter clockwise including in the transformations.
